## 3. Production Blockers

### P0

- Production-quality benchmark 不再只是 100-case synthetic foundation；v3.4 已补 155-case manually curated anonymized real-world-style benchmark path，v3.5 已补外部证明包结构，但真实 external provider run proof、外部人审和私有大样本证明仍缺。
- RAG 已有 local vector foundation、embedding persistence、semantic metadata、reranker contract、optional LLM grounded answer path 和 anonymized benchmark metrics，但缺真实 semantic provider 大规模验证、production-scale vector DB application path 和外部人审 groundedness/recall/citation certification。
- JD Parser、Resume Parser 已升级到 parser production foundation 和 optional LLM parser path，但默认仍是 deterministic local path；Match、Project Rewrite 已升级到 trustworthy foundation，但核心求职判断仍未达到生产完成。
- v3.0 已补应用层敏感字段加密，v3.4 已补 session list/revoke 和 backup purge/legal deletion template，但缺少 KMS/多 key 解密、rotation backfill、managed backup purge proof 和外部合规审计。
- Docker/Auth secret 风险：阶段 2.0 已禁止 Compose 空 secret，production 仍必须接入 secret manager、rotation 和环境隔离。

### P1

- Agent 已有 2.5 foundation 状态机，但仍缺 durable queue/worker、heartbeat、true async cancellation token、workflow lease 和 production observability。
- v3.3 已补主流程对象选择器和 minimal frontend gates，v3.4 已补 Playwright Chromium browser E2E；仍缺 deployed-backend browser E2E、accessibility、visual regression、cross-browser matrix 和 localStorage token hardening/httpOnly refresh rotation。
- PostgreSQL 不是生产默认，SQLite/local_data 仅适合 local dev。
- Observability 仍是 foundation：已有 structured request logging、request_id、readiness check、alert template、deployment proof template 和 v3.5 monitoring proof schema，但缺集中日志、metrics、tracing、alerting、eval trace pipeline 和 SIEM 的真实外部证明。
- Auth/Workspace 已有 route-level RBAC foundation、token revoke、session list/revoke 和 audit；仍缺 SSO/MFA、refresh token rotation/httpOnly cookie、advanced device controls、DB RLS 和 SIEM。

### P2

- Demo/synthetic/service-level foundation 数据和评测文档容易被误读为真实质量保障。
- 历史 release/acceptance 文档有“已完成/final”语境，需要持续以本 baseline 为准。
- Application/Interview/Study Plan 还没有真实外部系统、日历、通知、学习资源质量和运营审计。
